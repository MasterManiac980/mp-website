  • Created GitHub Rules and Roles

    Worked with Juno and Sophia to come up with the below roles and rules:

    Roles

    • Reviewer (New)
      • Role that will be given to anyone who makes a commit to COSI-Lab repositories and asks a current lab director to have the role assigned.
      • Members with the Reviewer role must be a current student at Clarkson University.
    • Current Member
      • Role assigned to any current members who would like to have write access to the COSI-Lab repositories.
    • Alumni
      • Role assigned to anyone who was at one point a member of the COSI-Lab organization and has since left Clarkson University.
    • Mirror
      • Role assigned to current Clarkson students who are also members of COSI-Lab and who actively participate in maintaining COSI's Mirror.

    Branch Rules

    • Main / Master
      • Changes to code on these branches requires a pull request
        • Pull requests require two approvals, by members with the reviewer role, before they can be merged into main
        • At least one review must be to the most recent commit on a pull request before it is allowed to be merged
      • Cannot push directly to these branches

  • Meetingbot

    • Written in C++ using D++ as the library for interacting with Discord
    • Sends a message in the #bot_reactions channel in the COSI Discord server asking for reactions
    • When someone reacts to the message, the bot assigns that user the @meeting_ping role, which allows Lab Directors to ping just those members who are interesting in getting meeting notifications so we dont have to do @everyone
